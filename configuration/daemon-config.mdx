title: "Daemon configuration"
description: "Reference for all fwupd.conf configuration options"
---

The fwupd daemon reads its main configuration file at startup. You can use it to control security policies, plugin behavior, update preferences, and hardware-specific settings.

## File location and format

The configuration file is located at:

```text
/etc/fwupd/fwupd.conf
```

<Note>
  The file may contain hardcoded or auto-generated credentials. Ensure it is readable only by the user running the fwupd process (typically `root`).
</Note>

The file uses an INI-style format with sections and key-value pairs:

```ini
[section]
key = value
```

- Lines beginning with `#` are treated as comments.
- Section and parameter names are case-sensitive.
- Boolean values may be `true` or `false` (case-insensitive).
- String values do not require quotes.

## [fwupd] section

The `[fwupd]` section controls core daemon behavior.

<ParamField body="DisabledDevices" type="string">
  A semicolon-delimited list of device GUIDs to block from being managed by fwupd. Use this to prevent specific hardware from being updated.
</ParamField>

<ParamField body="DisabledPlugins" type="string">
  A semicolon-delimited list of plugin names to block from loading. Run `fwupdmgr get-plugins` to see available plugin names.
</ParamField>

<ParamField body="ArchiveSizeMax" type="number">
  Maximum size (in megabytes) of a firmware archive that the daemon will load. Defaults to 25% of total system memory.
</ParamField>

<ParamField body="IdleTimeout" type="number" default="0">
  Number of seconds of idle time before the daemon shuts itself down. Set to `0` to never shut down automatically.

  <Note>Some plugins (such as thunderbolt) may inhibit auto-shutdown regardless of this setting.</Note>
</ParamField>

<ParamField body="IdleInhibitStartupThreshold" type="number" default="0">
  If the daemon takes longer than this value (in milliseconds) to start up, the idle shutdown timer is suppressed. Set to `0` to never suppress the timer.
</ParamField>

<ParamField body="VerboseDomains" type="string">
  Comma-separated list of log domains to enable verbose logging for. Leave unset to disable verbose logging for all domains. Set to `*` to enable verbose logging across all domains — equivalent to running the daemon with `--verbose --verbose`.
</ParamField>

<ParamField body="UpdateMotd" type="boolean">
  When `true`, fwupd updates the message of the day (MOTD) whenever device or metadata changes occur.
</ParamField>

<ParamField body="EnumerateAllDevices" type="boolean">
  When `false`, some plugins will only enumerate devices that are matched by available firmware metadata.
</ParamField>

<ParamField body="ApprovedFirmware" type="string">
  A list of firmware checksums approved by the site administrator. When set, only firmware whose checksum appears in this list can be installed. Leave unset to approve all firmware.
</ParamField>

<ParamField body="UriSchemes" type="string">
  Ordered list of URI schemes to use when downloading firmware. fwupd tries each scheme in order, falling back to the next if a download fails. Example: `https,http`.
</ParamField>

<ParamField body="IgnorePower" type="boolean" default="false">
  When `true`, fwupd ignores device power levels and proceeds with updates even when power is low.
</ParamField>

<ParamField body="IgnoreRequirements" type="boolean" default="false">
  When `true`, fwupd skips certain device requirements — for example, the generic GUID requirement for CHID, child, parent, or sibling devices.

  <Warning>Do not enable this on production systems. It is intended for firmware development use only.</Warning>
</ParamField>

<ParamField body="IgnoreEfivarsFreeSpace" type="boolean" default="false">
  When `true`, fwupd skips the free-space check on the EFI variable store for `db`, `dbx`, `KEK`, and `PK` updates. You may need this on Linux kernels older than 6.4 or on hardware that does not support `RT->QueryVariableInfo`.
</ParamField>

<ParamField body="OnlyTrusted" type="boolean" default="true">
  When `true`, fwupd only installs firmware signed with a trusted key.

  <Warning>Do not set this to `false` on a production or trusted system.</Warning>
</ParamField>

<ParamField body="ShowDevicePrivate" type="boolean" default="false">
  When `true`, fwupd exposes data such as device serial numbers, which some users may consider private.
</ParamField>

<ParamField body="TrustedUids" type="string">
  A list of Unix user IDs (UIDs). D-Bus callers whose UID matches an entry in this list are treated as trusted.
</ParamField>

<ParamField body="HostBkc" type="string">
  Comma-separated list of best-known-configuration (BKC) IDs used when running `fwupdmgr sync`. This can upgrade or downgrade firmware to reach a specific supported configuration level.

  Example: `vendor-factory-2021q1,mycompany-2023`
</ParamField>

<ParamField body="ReleaseDedupe" type="boolean">
  When `true`, duplicate releases with the same archive checksum are deduplicated across multiple sources.
</ParamField>

<ParamField body="ReleasePriority" type="string">
  Controls which copy of a release is preferred when the same firmware version is available from more than one source.

  | Value | Behavior |
  |-------|----------|
  | `local` | Prefer the locally cached copy to avoid a download. |
  | `remote` | Prefer the remote copy, which may have updated metadata such as release notes. |
  | _(empty)_ | No adjustment; ordering is determined by `OrderAfter`/`OrderBefore` in each remote. |
</ParamField>

<ParamField body="EspLocation" type="string">
  Explicit path to the EFI System Partition (ESP). Set this when UDisks cannot automatically detect the ESP location.
</ParamField>

<ParamField body="RequireImmutableEnumeration" type="boolean" default="false">
  When `true`, fwupd prevents plugins from directly interacting with devices during the probe or setup stages. All device information must come from sysfs or udev properties. Enabling this will block some plugins from functioning.
</ParamField>

<ParamField body="TrustedReports" type="string">
  Expressions that mark matching vendor reports as `trusted-report`. Each OR clause is separated by `;` and each AND clause by `&`.

  <Accordion title="Expression examples">

  | Expression | Meaning |
  |------------|---------|
  | `DistroId=chromeos` | Any report uploaded from ChromeOS. |
  | `DistroId=chromeos&RemoteId=lvfs` | Reports from ChromeOS machines in the `lvfs` remote. |
  | `DistroId=fedora&VendorId=19` | Reports from Fedora machines with vendor ID 19. |
  | `DistroId=fedora&VendorId=$OEM` | Reports from Fedora by the hardware OEM. |
  | `DistroId=fedora;DistroId=rhel&DistroVersion=9` | Reports from any Fedora version, or from RHEL 9. |
  | `Flags=is-upgrade,from-oem` | Reports that carry all specified flags. |

  The special value `$OEM` in `VendorId` refers to the OEM that owns the firmware. The os-release variables `$ID`, `$VERSION_ID`, and `$VARIANT_ID` may also be used in expressions.

  </Accordion>
</ParamField>

<ParamField body="P2pPolicy" type="string" default="nothing">
  Peer-to-peer sharing policy, used with an optional local caching service such as Passim. Reducing the number of redundant downloads can significantly lower network bandwidth usage.

  | Value | Behavior |
  |-------|----------|
  | `nothing` | Do not publish any files. |
  | `metadata` | Publish shared metadata only. |
  | `firmware` | Publish firmware archives after the next reboot. |

  Combine values with a comma, e.g. `metadata,firmware`.
</ParamField>

<ParamField body="TestDevices" type="boolean" default="false">
  When `true`, fwupd creates virtual test devices and a test remote. This is intended for CI testing and development only.
</ParamField>

## [uefi_capsule] section

The `[uefi_capsule]` section controls UEFI capsule update behavior.

<ParamField body="EnableGrubChainLoad" type="boolean" default="false">
  When `true`, fwupd configures GRUB to launch `fwupdx64.efi` directly, instead of using NVRAM or Capsule-on-Disk methods.
</ParamField>

<ParamField body="DisableShimForSecureBoot" type="boolean" default="false">
  When `true`, fwupd skips using the shim loader to chain-load the fwupd EFI binary. Only set this if `fwupd.efi` has been manually self-signed.
</ParamField>

<ParamField body="RequireESPFreeSpace" type="number">
  Minimum free space required on the ESP, in megabytes. For example, set `32` to require 32 MB. By default, fwupd dynamically requires at least twice the payload size.
</ParamField>

<ParamField body="DisableCapsuleUpdateOnDisk" type="boolean" default="false">
  When `true`, fwupd ignores the Capsule-on-Disk capability advertised by the firmware.
</ParamField>

<ParamField body="EnableEfiDebugging" type="boolean" default="false">
  When `true`, enables low-level debug logging from `fwupdx64.efi` into the `FWUPDATE_DEBUG_LOG` EFI variable.

  <Warning>
    This fills up the NVRAM store much faster. Enable only when debugging an issue with the EFI binary. This setting has no effect when using Capsule-on-Disk.
  </Warning>
</ParamField>

<ParamField body="RebootCleanup" type="boolean" default="true">
  When `true`, fwupd removes capsule files copied to the ESP and clears EFI variables set for the update after rebooting.

  <Note>Disable this only when debugging the flash process. Normal users should leave this enabled.</Note>
</ParamField>

<ParamField body="ScreenWidth" type="number">
  Override the screen width (in pixels) of the EFI framebuffer used by the UX capsule.
</ParamField>

<ParamField body="ScreenHeight" type="number">
  Override the screen height (in pixels) of the EFI framebuffer used by the UX capsule.
</ParamField>

## [thunderbolt] section

The `[thunderbolt]` section controls Thunderbolt plugin behavior.

<ParamField body="MinimumKernelVersion" type="string">
  Minimum Linux kernel version required before the Thunderbolt plugin will activate. You only need to change this on enterprise kernels that have back-ported Thunderbolt support to an older kernel version number.
</ParamField>

<ParamField body="DelayedActivation" type="boolean" default="false">
  When `true`, Thunderbolt firmware activation is delayed until the next shutdown, logout, or reboot.
</ParamField>

## [redfish] section

The `[redfish]` section provides credentials and connection settings for the Redfish plugin, which manages firmware updates for BMC and server hardware via the Redfish API.

<Warning>
  This section may contain credentials. Ensure `/etc/fwupd/fwupd.conf` is readable only by `root`.
</Warning>

<ParamField body="Uri" type="string">
  URI of the Redfish service in the format `scheme://ip:port`. Example: `https://192.168.0.133:443`.
</ParamField>

<ParamField body="Username" type="string">
  Username for authenticating with the Redfish service.
</ParamField>

<ParamField body="Password" type="string">
  Password for authenticating with the Redfish service.
</ParamField>

<ParamField body="CACheck" type="boolean" default="false">
  When `true`, fwupd verifies the Redfish server's TLS certificate against the system CA store. BMCs using self-signed certificates require this to be `false`.
</ParamField>

<ParamField body="IpmiDisableCreateUser" type="boolean" default="false">
  When `true`, fwupd will not use IPMI KCS to automatically create an initial user account on the BMC when no SMBIOS data is available.
</ParamField>

<ParamField body="ManagerResetTimeout" type="number">
  Number of seconds to wait for the BMC to restart after a manager reset operation.
</ParamField>
