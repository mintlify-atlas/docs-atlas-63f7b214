`fwupdmgr` is the command-line client for fwupd. It communicates with the fwupd daemon over D-Bus and lets you discover devices, check for firmware updates, apply them, and report results — all from a single tool.

<Note>
  Install fwupd using your distribution's package manager rather than building it from source. Your distro's package maintainer has already tested fwupd against your system's software and configuration. Building from source is only necessary for plugin development or debugging.
</Note>

## Update your firmware

<Steps>
  <Step title="Check what devices fwupd can see">
    List all devices that fwupd has detected on your system:

    ```bash
    fwupdmgr get-devices
    ```

    Each entry shows the device name, current firmware version, and whether it is updatable.
  </Step>

  <Step title="Download the latest metadata">
    Fetch the current firmware catalog from LVFS:

    ```bash
    fwupdmgr refresh
    ```

    This downloads the latest metadata from configured remotes so fwupd knows what updates are available.
  </Step>

  <Step title="Check for available updates">
    Compare the metadata against your installed firmware versions:

    ```bash
    fwupdmgr get-updates
    ```

    If updates are available for any device on your system, they are listed here with version information and release notes.
  </Step>

  <Step title="Apply all updates">
    Download and install every available update:

    ```bash
    fwupdmgr update
    ```

    fwupd applies each update using the appropriate method for that device:

    - **Live updates** are applied immediately without a reboot.
    - **Staged updates** are written to a staging area and applied at the next system reboot.

    Follow any on-screen prompts, and reboot if instructed.
  </Step>
</Steps>

## Report update results to LVFS

After updating, you can send a report of successful and failed updates back to LVFS. This is optional but encouraged — it gives firmware vendors and LVFS administrators data on real-world update success rates.

```bash
fwupdmgr report-history
```

Only updates distributed through LVFS are included in the report. You can review the [LVFS privacy policy](https://lvfs.readthedocs.io/en/latest/privacy.html) before reporting.
